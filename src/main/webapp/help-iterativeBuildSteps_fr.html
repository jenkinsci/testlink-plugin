Le Plug-in TestLink Jenkins récupère les données de cas de test automatisés à partir de TestLink. Ces données sont ensuite utilisées en variables d'environnement. Ainsi vos Build Steps itératives peuvent utiliser ces informations pour exécuter vos tests automatisés.

Par exemple, vous pourriez exécuter ant en passant une Test Class en paramètre: 

/opt/java/apache-ant-1.8.0/bin/ant -DtestClass=$TESTLINK_TESTCASE_TEST_CLASS

Liste des variables d'environnement que le Plug-in TestLink Jenkins défini:

  • TESTLINK_TESTCASE_ID
  • TESTLINK_TESTCASE_NAME
  • TESTLINK_TESTCASE_TESTPROJECTID
  • TESTLINK_TESTCASE_AUTHOR
  • TESTLINK_TESTCASE_SUMMARY
  • TESTLINK_BUILD_NAME
  • TESTLINK_TESTPLAN_NAME
  • TESTLINK_TESTCASE_$CUSTOM_FIELD_NAME

Pour chaque champ personnalisé trouvé, le Plug-in TestLink Jenkins défini une variable d'environnement avec ses nom et valeur, comme indiqué ci-dessus (%CUSTOM_FIELD_NAME%). Si le nom du champ personnalisé contient des espaces, the plug-in va les convertir en _.
